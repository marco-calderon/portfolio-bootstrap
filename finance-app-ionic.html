Finance Mobile App

Repository  Demo 
Angular Ionic

A finance mobile app style implemented into Ionic Framework for iOS and Android devices, and potentially for web, that offers the user to manage their savings and expenses while having the information at hand at all times.

It also gives insights and statistics of your savings and expenses in tables and categories so you will have them organized.

The mobile app is made using Ionic Framework v5 and Angular 11. It is structured to obtain data from a service although is not yet implemented, but the models exists to adapt the data for the existing components. It also has implemented a storage mechanism to save the information to the persistence state of the device instead of using localStorage (when in web it fallbacks to IndexedDB).

The scope of this app was to develop the application for iOS devices only, but it fully supports Android and web, and can be found in the link above or by clicking here.

The CSS was not provided for this application, so styles should be created to conform to the design. The library used for showing charts is Chart.js as it is very straightforward to use and allows configuration of the data and colors that are displayed on it. The donut chart used here shows sample data that could be easily obtained from an external service - the application is structured for requesting data so it would be simple to attach an external REST API for this to save the data to the cloud.

It implements the Sandbox pattern, in which the application uses a shared service across the components that obtain data independently of the location of it - could be retrieved locally from the device or obtained from a service. This is the main purpose of this pattern, because the back end was not provided in this case but it can be plugged in without too much hussle.

              
  @Component({
    selector: 'app-analytics',
    templateUrl: './analytics.page.html',
    styleUrls: ['./analytics.page.scss'],
  })
  export class AnalyticsPage implements OnInit {
    categories: CategoryModel[];
  
    constructor(private sb: SandboxService) { }
  
    ngOnInit() {
      const chartData = this.sb.getChartData();
      this.initChart(chartData.labels, chartData.data);
  
      this.categories = this.sb.getCategories();
    }
  }

In Angular, it is also important to follow some criteria when building applications to ensure that code footprint is minimized, performance is increased and reusability is primordial. With this in mind, the application’s architecture was made based on Components / Containers / Pages, with their own styles, so the Components are the basic piece of the app, only containing stateful behavior when needed and not modify the app state internally. They are used by Containers, which they can request data and pass down to components. Lastly, Pages use Containers for structuring the data inside every screen that is shown on the device.

This pattern ensures that there are no side effects made by Components (since it prevents the application to be tested successfully) and it can lead to errors and unexpected behavior. In the same way, Containers can be tested against the data independently of the Components that they render.

This also introduces new benefits for mobile and web applications - they can be lazy loaded, tested with tools such as Karma and Storybook, and sepparates the styles from components that are not related. This way we can build fully extensible and scalable apps that are performant and resilient to changes because the logic that lies behind of the project exists only on the parts that are important, similar to the separations of concerns principle of SOLID.

One of the challenges of every mobile and web application is confront the changes with the least time possible and avoid making changes on places that are not related to it. In a team context, this also means that every developer can understand the changes and the files normally don’t conflict with other’s work - thus leading to a faster development environment and less stress and frustration for the team player.
